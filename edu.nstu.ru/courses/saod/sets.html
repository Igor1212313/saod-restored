Множество  - набор элементов, между которыми существуют структурные отношения или отношения порядка.

 Множества делятся на позиционные и ассоциативные. 

 В позиционных множествах  доступ к элементу выполняется по предопределенной или указанной позиции. К позиционным множествам можно отнести стеки, очереди, списки, графы. К ассоциативным множествам относятся  очередь с приоритетом, дерево поиска, хеш - таблица.

Стеки и очереди  – динамические множества, в которых элемент, удаляемый из множества, задается не произвольно, а определяется структурой. Из стека можно удалить только тот элемент, который был добавлен в него последним. Стек работает по принципу “последним пришел – первым ушел” (LIFO – Last  In - First Out). Напротив, из очереди  можно удалить только тот элемент, который находился в очереди дольше всего. Очередь работает по принципу “первым пришел – первым ушел” (FIFO – First  In -  First  Out). 

Списки предназначены для хранения множеств - последовательностей элементов. Если местоположение элемента определяется указанной позицией, то список можно отнести к позиционному множеству. Если местоположение в списке определяется значением элемента, то список  является ассоциативным множеством.

В ассоциативных множествах  местоположение элемента определяется его значением или значением  ключа. Как правило, элемент ассоциативного множества представляет собой  запись, содержащую различные поля. Одно из полей рассматривается как ключ, предназначенный для идентификации элемента среди прочих элементов. Остальные поля содержат дополнительные данные. Элемент в множестве ищется по ключу, когда он найден, можно прочесть или изменить дополнительные данные. 

Если ассоциативное множество допускает существование нескольких элементов с одинаковым ключом. то оно называется мультимножеством.

Позиционные и ассоциативные множества хранятся в программных структурах данных, которые строятся в виде массивов элементов или  множества  элементов, связанных между собою адресными указателями(связных структур).

Достоинством структур данных на базе массивов является прямой доступ к элементам по указанной позиции с помощью операции индексирования, отсутствие дополнительных затрат памяти на связи элементов.  Но, структура на базе массива, как правило, может хранить ограниченное размером массива  множество элементов, не изменяющее количества элементов.

Напротив, связная структура на базе адресных указателей размещает элементы в динамической  памяти по мере включения в множество. Структура не накладывает принципиальных ограничений на количество элементов множества.

Недостатком связных структур является последовательный доступ к элементам, дополнительные расходы памяти на организацию связей элементов через адресные указатели.

Операции над множествами.

Развитые структуры, предназначенные для хранения изменяющихся множеств, называются динамическими. Разные алгоритмы используют разные операции над множествами. Например, требуется лишь добавлять и удалять элементы в множество, а также проверять, принадлежит ли данный элемент множеству. Структура данных, поддерживающая такие операции, называется словарем.  Другая структура, называемая очередью с приоритетами,  разрешает выбирать и удалять элемент с наименьшим значением приоритета.

Операции делятся на запросы, не меняющие множества, и операции, меняющие его.

Типичные операции над множествами следующие:

Search (S, k) (поиск) - запрос, который по данному ключу k возвращает указатель на элемент множества S c ключом k . Если такого элемента в множестве нет, возвращается nil (NULL для адресных указателей или – 1 для индексов).

Insert (S, x) – оператор добавляется к множеству S элемент, на который указывает указатель х. К этому моменту все поля в записи элемента заполнены.

Delete (S, x) – оператор удаляет из множества S элемент, на который указывает указатель х.

Minimum (S) – запрос возвращает указатель на элемент множества S с наименьшим ключом (считается, что ключи линейно упорядочены).

Maximum (S) – запрос возвращает указатель на элемент с максимальным ключом.

Successor (S, x) – запрос возвращает указатель на элемент множества S, непосредственно следующий за элементом х в смысле линейного порядка ключей. Если х указывает на наибольший элемент, то будет возвращен NIL.

Predecessor (S, x) – запрос возвращает указатель на элемент множества S, непосредственно предшествующий элементу х. Если х – наименьший элемент, возвращается nil.
