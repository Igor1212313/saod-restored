При сортировке вставками каждый очередной элемент помещается в надлежащее место среди отсортированных элементов. Для освобождения места для вставляемого элемента производится смещение части элементов в отсортированной части на одну позицию вправо.

Тест

 Сортировка включениями имеет показатель эффективности О(n²). Но в отличие от сортировки выбором это адаптивный алгоритм, поскольку число повторений внутреннего цикла зависит от значения ключа включаемого элемента. Если массив изначально упорядочен, то внутренний цикл вообще не выполняется. В этом эффективность алгоритма максимальна и оценивается нотацией О(n). 

  Приведенный алгоритм можно ускорить, если убрать из внутреннего цикла проверку j>0. Для этого перед сортировкой нужно найти в массиве минимальный элемент и переместить его в начало массива.

Insertion_Sort (A [1..n ]) 

  1. A [1 .. n ] – массив размерностью n

  2.      for i  ← 2 to n 

  3.                  do x  ←   A [i]  

  4.                         j ←  i - 1 

  5.                       while j>0 and A [j] >  x 

  6.                               do A [j+1]  ←  A [j] 

  7.                                      j ←  j - 1 

  8.                       A [j+1]  ← x  

  9.      return

  Ещё одно усовершенствование - применить бинарный поиск места включения в отсортированной части. Такой алгоритм называется сортировкой бинарными включениями:

Bin_Insert_Sort (A [1..n ])

  1. for i ← 2 to n

  2.          do x  ←  A [j]   

  3.                  l ← 1

  4.                   r  ← i - 1

  5.                   while l <= r

  6.                      do    m  ←  (ll+ r /2

  7.                                   if  x < A [m] 

  8.                                          then r  ← m - 1

  9.                                          else ll←  m+1

  10.                    for   i  ← j - 1 down l

  11.                           do A [j+1]  ←  A [j]

  12.                     A [j]  ←  x   

  13.  return

  К сожалению, это улучшение уменьшает только количество операций сравнения до оценки О(log n), но не влияет  на количество операций сдвига. Поскольку сдвиги ключей и сопутствующих данных требует значительно больше времени, то это улучшение не является решающим. Общая оценка эффективности по прежнему  равна О(n²).

  Этот пример показывает, что очевидное улучшение часто оказывается немного менее существенным, чем ожидалось.

  В конечном счете сортировка включениями оказывается не очень подходящим методом для вычислительных машин, поскольку сопровождается не экономичным сдвигом ряда элементов на одну позицию. Лучших результатов следует ожидать от метода, при котором пересылки выполняются для отдельных элементов и на большие расстояния.
