Шейкер-сортировка является усовершенствованным методом пузырьковой сортировки. Во-первых, вводится досрочное прерывание проходов, если на очередном проходе обнаруживается, что файл уже отсортирован. Для этого на каждом проходе запоминается, производился ли обмен на этом проходе.

   Во-вторых, если запоминать не только факт обмена, но и место (индекс) последнего обмена, то можно ограничить просмотр пар на следующем проходе этим индексом. Действительно, поскольку левее этого индекса не было обменов, значит все пары левее этого индекса уже упорядочены и нет смысла их просматривать.

    Алгоритм пузырьковой сортировки демонстрирует асимметрию : один неправильно расположенный "легкий" элемент всплывает на положенное место за один проход,  а неправильно расположенный "тяжелый" элемент  опускается за один проход на одну позицию в сторону правильного места.

  Например, массив 12   18   42   44   55   67   94   06  будет отсортирован за один проход, а массив 94   06   12   18   42   44   55   67 будет отсортирован за 7 проходов.

   Эта асимметрия подсказывает третье улучшение - менять направление следующих один за другим проходов. Поэтому алгоритм называется шейкер - сортировкой.

   Для массива 94   06   12   18   42   44   55   67 вместо 7 проходов шейкер - сортировка дает 5 проходов причем на каждом проходе количество просматриваемых пар уменьшается.

Sheiker_Sort ( A [1..n ] ) 

1.     A[1 .. n] – массив размерностью n

2.     l ← 1

3.     r ← n

4.     while  l < r

5.               do  for   j ← r  down l +1

6.                           do  if  A[j-1] > A[j]

7.                                    then  temp ← A[j-1]

8.                                             A[j-1] ← A[j]

9.                                             A[j] ← temp

10.                                           k ← j

11.                  l ← k

12.                  for   j ← l to r

13.                        do  if  A[j-1] > A[j]

14.                                 then temp ← A[j-1]

15.                                           A[j-1] ← A[j]  

16.                                           A[j] ← temp

17.                                           k ← j

18.                  r ← k-1

19.   return

   Анализ алгоритма Шейкер-сортировки довольно сложен. Кнут утверждает, что среднее число проходов пропорционально n- k*√n и среднее число сравнений пропорционально ½ (n² - n(k+ln n)). Но число обменов, как и в методе пузырька равно ¼ (n² -n) [7].

  Эмпирические исследования показали, что алгоритм шейкер - сортировки выгодно использовать, когда элементы файла почти упорядочены.
